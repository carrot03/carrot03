🦀 Backend — main.rs

Key functions

🔍
symbol_to_coin()
Maps a ticker string to its CoinGecko ID, display name. Case-insensitive linear scan over the COINS const array.
🌐
fetch_json()
Generic async HTTP GET using reqwest. Returns serde_json::Value or a (StatusCode, String) error.
📊
api_coins()
Fetches all 20 coins in one CoinGecko call, passes the JSON array straight through to the client.
🪙
api_coin()
Resolves symbol → CoinGecko ID, fetches single-coin market data, returns first element of the array.

🖥️ Frontend — index.html

Single embedded file, compiled into the binary via include_str!. No build step, no bundler, no framework.

📊 Global Stats Bar
Total market cap + 24h change, volume, BTC & ETH dominance, active coin count. Sourced from /api/global.
📋 Sortable Table
Click any column header to sort ascending/descending. Columns: rank, name, price, 1h%, 24h%, 7d%, market cap, volume, sparkline.
📈 SVG Sparklines
7-day price chart per row rendered as an inline SVG polyline. Green or red based on the 7d change direction. No Chart.js overhead per row.
🔍 Search & Filter
Real-time filter by coin name or ticker symbol. Works in combination with the Watchlist filter.
⭐ Watchlist
Star any coin row to add it to your watchlist. Toggle button shows only favourites. Persisted in localStorage.
🪟 Detail Modal
Click a row for price, 1h/24h/7d changes, market cap, volume, ATH, circulating supply, max supply, and a Chart.js 7-day line chart.
🌙 Dark / Light Theme
CSS custom properties drive the full colour scheme. Toggle in the header, persisted in localStorage.
🔄 Auto-Refresh
Data refreshes every 60 seconds. A countdown timer is shown in the header. Both /api/coins and /api/global are fetched in parallel.

☁️ Deployment

🐳
Multi-stage Docker build
Stage 1 rust:1.86-slim — compiles the release binary with dependency caching. Stage 2 debian:bookworm-slim — copies only the binary + CA certs. Final image: ~27 MB.
✈️
Fly.io — crypto-api-htantawi.fly.dev
Region: iad (Washington DC). 2 machines for HA. 256 MB RAM, 1 shared vCPU. Auto-stops when idle; auto-starts on first request.
🔒
HTTPS enforced
Fly.io terminates TLS. force_https = true in fly.toml. Server listens on 0.0.0.0:$PORT.

Recommended Test Plan

⚡ Phase 1 — Unit tests (Rust) · already in place ✓ Done
  • Run: cargo test
  • File: src/main.rs — #[cfg(test)] mod tests
  • Tests the only pure logic in the backend: symbol_to_coin(). No I/O, no async, no mocking needed.
🔗 Phase 2 — Handler tests (in-process, mocked HTTP) ✗ Not started
  • Add crate: axum-test = "15" (dev-dependency) + wiremock = "0.6"
  • Pattern: boot the Axum router in-process, spin up a wiremock server to fake CoinGecko, fire HTTP requests, assert on status codes and JSON fields.
  • Cases to cover: GET /api/coin/btc → 200 JSON with symbol == "btc"; GET /api/coin/zzz → 400; GET /api/coins → 200 array length == 20; CoinGecko returns 429 → server returns 502.
  • No network calls: wiremock intercepts all outbound requests — tests run fully offline.
🌐 Phase 3 — Contract tests (live CoinGecko, skipped in CI) ✗ Not started
  • Marked #[ignore] so they don't run on every cargo test. Run manually with cargo test -- --ignored.
  • Assert the actual CoinGecko response still contains the fields the app depends on (current_price, sparkline_in_7d, etc.). Catches upstream breaking changes early.
  • Rate-limit friendly: one call per test, no parallelism.
📐 Phase 4 — JS formatter unit tests ✗ Not started
  • Extract fmt.price(), fmt.big(), fmt.pct(), and sparkline() into a separate src/utils.js file.
  • Test with Vitest or plain node --test (no framework needed for pure functions).
  • Key edge cases: null → "—"; 0; values > 1T; negative prices; empty sparkline array.

Quickstart — adding a handler test

// In Cargo.toml: // [dev-dependencies] // axum-test = "15" // wiremock = "0.6" // tokio = { version = "1", features = ["full"] } #[cfg(test)] mod handler_tests { use axum_test::TestServer; use wiremock::{MockServer, Mock, ResponseTemplate}; use wiremock::matchers::path_regex; #[tokio::test] async fn unknown_symbol_returns_400() { let server = TestServer::new(build_app()).unwrap(); let resp = server.get("/api/coin/zzz").await; resp.assert_status_bad_request(); } #[tokio::test] async fn coin_endpoint_returns_json() { let mock_cg = MockServer::start().await; Mock::given(path_regex(r"/api/v3/coins/markets")) .respond_with(ResponseTemplate::new(200) .set_body_json(serde_json::json!([{"id": "bitcoin", "symbol": "btc"}]))) .mount(&mock_cg).await; // inject mock_cg.uri() into the app as the base URL let resp = TestServer::new(build_app_with_base(mock_cg.uri())) .unwrap().get("/api/coin/btc").await; resp.assert_status_ok(); resp.assert_json_contains(&serde_json::json!({"symbol": "btc"})); } }

⚠️ Known Limitations

No caching. Every frontend refresh triggers live CoinGecko API calls. Under heavy traffic this will hit the free tier rate limit (~30 req/min).
  • No server-side cache — a simple in-memory TTL cache (e.g. Arc<Mutex<CachedData>>) would fix this.
  • Panics on server startup (.unwrap() in main()) — acceptable for a learning project; a production app would use structured error handling.
  • No rate-limit handling — CoinGecko 429 responses are passed through as 502 Bad Gateway.
  • Coin list is hardcoded — adding a coin requires a code change and redeploy.
  • No tests for HTTP handlers — only pure logic (symbol_to_coin) is unit-tested. Handler tests would require axum-test or tower::ServiceExt.
